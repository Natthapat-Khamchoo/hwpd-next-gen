# HWPD Next Gen — Standalone Web Application (Next.js)

ระบบต้นแบบรายงานและควบคุมปฏิบัติการ ตำรวจทางหลวง (HWPD Next Gen) ที่ได้รับการย้ายฐานการทำงาน (Migrate) จาก Google Apps Script (GAS) มาเขียนเป็นแอปพลิเคชันเดี่ยว (Standalone Web Application) เพื่อความยืดหยุ่น ประสิทธิภาพ และการทำงานที่ไม่ติดขีดจำกัดด้านความเร็วของ GAS

---

## 🛠️ เทคโนโลยีที่ใช้ (Tech Stack)

1. **Frontend & Backend Server:** **Next.js 15+ (App Router)** + **TypeScript**
   - ใช้สถาปัตยกรรมแบบ Full-Stack ในโฟลเดอร์เดียว โดยฝั่ง API จะอยู่ที่ `/api/...` และหน้าบ้านจะเป็น React Component
2. **Database ORM:** **Prisma**
   - ช่วยในการจัดการฐานข้อมูลอย่างเป็นระบบ รองรับทั้งการทำ Schema Migration และการเขียนคำสั่ง Query แบบ Type-Safe
3. **Database (ฐานข้อมูลพัฒนา):** **SQLite (dev.db)**
   - ถูกกำหนดค่าเริ่มต้นไว้เป็น SQLite เพื่อให้ผู้ใช้สามารถทดลองรันระบบในเครื่องตนเองได้ทันทีโดยไม่ต้องติดตั้ง PostgreSQL หรือตัวจัดการฐานข้อมูลภายนอก (เมื่อนำไปใช้งานจริงบน Production เพียงแค่แก้ไขไฟล์ `schema.prisma` เปลี่ยน Provider เป็น `postgresql` เท่านั้น)
4. **Authentication:** **JSON Web Tokens (JWT)** และการเข้ารหัสความปลอดภัยด้วย **bcryptjs**

## 📁 โครงสร้างโปรเจกต์ (Project Structure)

```bash
hwpd-next-gen/
├── prisma/
│   ├── dev.db             # ฐานข้อมูล SQLite (สร้างขึ้นหลัง migrate)
│   ├── schema.prisma      # โมเดลการจัดเก็บข้อมูลทั้งหมดของตำรวจทางหลวง
│   └── seed.ts            # สคริปต์เพิ่มบัญชีผู้ใช้ทดสอบและข้อหาทางกฎหมาย
├── src/
│   ├── app/
│   │   ├── api/           # โฟลเดอร์หลังบ้าน (Backend API Endpoints)
│   │   │   ├── auth/      # API เข้า/ออกจากระบบ และเช็คเซสชัน (/login, /logout, /me)
│   │   │   ├── admin/     # API สำหรับแอดมินอนุมัติข้อมูล (/pending, /action)
│   │   │   └── reports/   # API สำหรับบันทึกรายงานต่าง ๆ ของสายตรวจ (/reports)
│   │   ├── login/         # หน้าจอเข้าสู่ระบบแบบ Glassmorphic (หน้าบ้าน)
│   │   ├── globals.css    # ธีมสี ตกแต่งปุ่ม และการไล่เฉดสีสุดพรีเมียม
│   │   ├── layout.tsx     # เลย์เอาต์หลักที่โหลดฟอนต์และไอคอนภายนอก
│   │   └── page.tsx       # เลย์เอาต์และเมนูควบคุมการใช้งานแยกตามสิทธิ์ผู้ใช้
│   └── lib/
│       └── prisma.ts      # ไคลเอนต์เชื่อมต่อฐานข้อมูลตัวกลาง (Prisma Singleton)
├── package.json           # การตั้งค่า dependencies ของระบบ
└── tsconfig.json          # การตั้งค่าตัวแปลภาษา TypeScript
```

## 🚀 วิธีการทดสอบและรันระบบในเครื่อง (Getting Started)

1. **ติดตั้ง Dependencies:**
   ```bash
   npm install
   ```

2. **สร้างฐานข้อมูลและเพิ่มข้อมูลตัวอย่าง (ทำเฉพาะครั้งแรก):**
   ```bash
   npx prisma migrate dev --name init
   ```
   *(คำสั่งนี้จะสร้างไฟล์ฐานข้อมูล `prisma/dev.db` และดำเนินการเพิ่มบัญชีผู้ใช้งานตัวอย่างให้โดยอัตโนมัติ)*

3. **รันเซิร์ฟเวอร์สำหรับพัฒนา (Development Server):**
   ```bash
   npm run dev
   ```

4. **เปิดเบราว์เซอร์เข้าใช้งาน:**
   - URL: [http://localhost:3000](http://localhost:3000)
